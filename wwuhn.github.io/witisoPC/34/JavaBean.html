JavaBean

JavaBean可以实现代码重用，以及使业务逻辑，存取数据等操作和JSP页面代码分离，从而可以使整个系统结构更加清晰，虽然JavaBean可以部分地实现控制层（control)和视图层（view)的相互独立，但它并没有完全实现MVC开发模式；

将一些逻辑操作独立出来写到JavaBean类中，然后在JSP文件中调用从而使得JSP文件更多地只负责数据显示功能；

JavaBean的使用实现了业务逻辑和视图层的分离，使得代码的重用性提高，另外也分化了工作职责，JSP自带的标准动作元素和标签已简化了JSP页面的开发过程，同时JSP技术允许用户自定义标签，用它来封装一些动态功能，而JSTL(Java Server Pages Staandard Tab Library)抽取了最为常用的标签操作。JSTL可以部分地取代传统JSP程序中嵌入Java代码的做法。

使用JavaBean注意事项

1 JavaBean一定要有包如package ch8;

2 JavaBean一定要有公共public类;

3 JavaBean要有构造函数;

4 用Jcreate编译,生成的包要放入项目/WEB-INF/classes/

JavaBean的属性都设置为private,怎么读写呢?设置set和get方法,如果只单独设置set属性,表示这个属性只能写,如果get,则只能读;

JavaBean既然是一个类，就会有属性，一般建议将这些属性封装起来供其他JSP或Servlet来存取。JavaBean一旦封装后，JSP就可以搭配Action标签<jsp:setProperty>与<jsp:getProperty>来存取属性值；但<jsp:setProperty>属性名称必须与窗体的组件变量名称相同，方可方便从窗体中取得用户输入的值后直接存入JavaBean的属性内；否则必须使用<jsp:setProperty>的属性param来设定。

为了更快速地设定所有JavaBean属性，只要窗体组件变量名称都与JavaBean的属性名称一致，就可以使用<jsp:setProperty property="*">方式来设定所有JavaBean属性。
