数据结构(Data Structure)由数据和结构两部分组成。其中，数据部分是指数据元素的集合；结构就是关系，结构部分是指数据元素之间关系的集合。所以，数据结构就是指数据元素的集合及数据元素之间关系的集合。概括地讲，数据结构就是指相互之间有一种或多种特定关系的数据元素的集合。在计算机上要处理数据，就要保存数据及它们之间的关系。

1 数据的逻辑结构

数据的逻辑结构(Logic Structure是从具体问题抽象出来的数学模型，与数据在计算机中的具体存储没有关系。数据的逻辑结构独立于计算机，是数据本身所固有的特性。从逻辑上可以把数据结构分为线性结构和非线性结构，主要包括：集合、线性、树形和图状结构。

1.1 集合结构

集合关系(set Structure)中的元素“同属于一个集合”，如整数集、字符集。结构体也可以看作是一个集合。

1.2 线性结构

线性结构(Linear Structure)中的数据元素之间存在“一对一”的关系，比如数组、队列等。

1.3 树形结构

树形结构(Tree Structure)中的数据元素之间存在“一对多”的关系。如电脑硬盘存储的文件系统的文件夹与文件的关系、公司的组织结构图等。

1.4 图状结构

图状结构(Graphic Structure，也称网状结构)中的数据元素之间存在“多对多”的关系。比如城市交通图等。

树形结构和图状结构又称为非线性结构。由于集合中数据元素的关系是非常松散的，因此，常用其他几种结构来描述集合。

（数据类型表明数据元素是否“同类”的关系，是在程序设计中已经实现了的数据结构，类型显式或隐式地规定了数据的取值范围、存储方式以及允许进行的运算。数据类型分为不可分解的原子类型及可分解的结构类型。比如，C语言中的整型、实型、字符型就属于原子类型；而数组、结构体和共用体类型就属于结构类型，由其他类型构造得到。）

数据结构由相互之间存在着一种或多种关系的数据元素的集合和该集合中数据元素之间的关系组成。数据结构的形式化定义记为：Data_Structure = (D,R)。

其中，D是数据元素的有限集合，R是数据集合D中所有元素之间的关系的有限集合。

需要注意的是，即使是由相同元素构成的集合，只要定义的关系不同，也不是同一数据结构。

数据结构可以分为逻辑上的数据结构和物理上的数据结构。数据结构的形式化定义为逻辑结构。物理结构为数据在计算机中的存储，它包括数据元素的存储和关系的存储。如图状结构的数据元素是顶点，可以用一个一维数组来存储，每个顶点的边（与其它顶点的直接、间接、连接或没有连接，也就是数据元素的关系）也可以用一个一维数组来存储，如果有n个顶点，则有n个有n个元素的一维数据，也就是一个n行n列的二维数组，也称为方阵或矩阵。

2 数据的物理结构(Physical Structure)

数据的物理结构，又称为存储结构(Storage Structure)，是指数据的逻辑结构在计算机存储空间的存储存放形式。数据的物理结构是数据结构在计算机中的存储，它包括数据元素的机内存储和关系的机内存储。由于具体实现的方法有顺序、链式、索引、散列等多种，因此，一种数据结构可存储成一种或多种存储结构。

（逻辑关系可以用存储关系来描述，也可以用特别定义的数据来描述，如图的邻接矩阵。）

数据元素的机内存储方法：用二进制位的位串存储数据元素。通常称这种位串为节点(Node)。当数据元素由若干个数据项组成时，位串中与各数据项对应的子位串称为数据域(data field)。因此，节点是数据元素的机内存储。

关系的机内存储：数据元素之间的关系的机内存储可以分为顺序存储和非顺序存储，这样就可以得到两种不同的数据结构，即顺序存储结构和链式存储结构。顺序存储结构借助元素在存储器中的相对位置来存储数据元素之间的逻辑关系。链式存储借助指示元素存储位置的指针(pointer)来存储数据元素元素之间的逻辑关系。

2.1 顺序存储结构(Sequence Storage Structure)

顺序存储结构通过数据元素在计算机存储器中的相对位置来存储数据元素的逻辑关系，一般把逻辑上相邻的数据元素存储在物理位置相邻的存储单元中，它是一种最基本的存储方法，一般采用数据来实现。

2.2 链式存储结构(Linked Storage Structure)

链式存储结构对逻辑上相邻的数据元素不要求其存储位置必须相邻，元素间的逻辑关系通过指针来存储，一般采用链表来实现。链式存储结构中的数据元素称为节点(node)，在节点中附设地址域(Address Domain)来存储与该节点相邻的节点的地址。从而实现节点间的逻辑关系。

3 数据的运算

算法的设计取决于数据的逻辑结构，而算法的实现依赖于数据采用的存储结构。数据的存储结构实质上是它的逻辑结构在计算机存储器中的实现，为了全面反映一个数据的逻辑结构，它在存储器中的映象包括两个方面的内容。即数据元素之间的信息和数据元素之间的关系。不同的数据结构有其相应的若干运算。数据的运算是在数据的逻辑结构上定义的操作算法，如检索、插入、删除、更新、排序、求长度、初始化、创建等。数据的运算是数据结构的一个重要方法。讨论任何一种数据结构都离不开对该结构中元素的检索、插入、删除、更新、排序的操作的探讨。

4.1 线性表的顺序存储

typedef int ElemType //定义元素类型为int

typedef struct(){

4.2 线性表节点的单链表存储

typedef struct LNode{

ElemType data;

}LNode, *LinkList;

4.3 用数组实现栈中元素的存储

#define MAXSIZE 100 //定义一个静态顺序栈的最大长度

typedef struct{

SElemType elem[MAXSIZE]; //定义一个存放栈数据元素的一维数组

int top; //栈顶指针，栈非空时始终指向栈顶元素的下一位置

int StackSize; //定义一个变量，存储顺序栈的当前长度

}SeqStack;

4.4 顺序队列

#define MAXSIZE 100 //定义顺序队列的最大容量

typedef struct{

在面向过程的程序设计中，数据结构都是具体的，形式上没有规定各个过程和数据结构的关系。而面向对象的程序设计的基本思想是数据结构抽象化。在定义数据结构时，就要做好两方面的规定。第一，数据结构的性质、状态或属性；第二，规定在此数据结构上允许实行的操作。凡是没有事先规定的操作以后都不允许使用，这样的数据结构称为一个对象或一个“类”。从形式上看，数据结构与操作的关系不自由了，然而，这些对象是抽象地定义了，运行中可以随时形成新的、同类的数据结构。程序的运行就是动态地激活和调用各种对象。对象的调用顺序和同时激活的对象数目都是相当自由的。
