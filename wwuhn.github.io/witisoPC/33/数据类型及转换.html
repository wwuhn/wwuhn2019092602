数据类型及转换

要将数据类型转换为其它类型，可在变量前指定新类型，然后用括号括起来，在类型转换中，如果目标类型的取值范围比源类型大，转换会很容易，因为新的类型将有足够的空间来存储它。

有时不用进行类型转换即可直接使用类型不同的变量，如char变量可直接用作int变量，int可直接用作long变量，任何类型可直接用作double变量。

Java语言定义了8种基本数据类型，而引用类型则是用户自行定义的，所以这的数量是无限的。

基本数据类型需要0－8个字节的内存空间，当一个基本数据值存储于变量或传递给一个method时，计算机会将它们所拥有的值复制一份，object就需要更多的内存了，object存储于内存，当此object被创建时会在堆栈中动态地分配其位置，同时在此object不需要使用时，它的内存就会自动地被回收，当object被指定给一个变量或被传递给一个method时，则该object就不需要被复制了，而是只引用存储该变量或该method的内存。

在Java中，处理基本数据类型的值，是把数据取出来操作，而处理object值，则是通过引用的方式来操作。

声明基本数据类型的变量时，系统同时为该变量分配存储空间，此空间将直接保存基本数据类型的值；

在声明引用变量时，系统只为该变量分配引用空间；

在简单变量和对象之间进行转换

不能在对象和简单变量之间进行类型转换，在Java中，这些信息类型差别太大，无法相互替代；

在Java lang包中，有与每种简单变量类型对应的类Boolean,Byte,Character,Double,Float,Integer,Long,Short;

使用这些类中定义的类方法，可以将变量的值作为参数来创建一个对象，如：

Integer suffix=new Integer(5309);

int newSuff=suffix.intValue();

String count="25";

int myCount=Integer.parseInt(count);

基本数据类型与wrapper类

基本数据类型单纯，所以占用内存空间小，计算速度也比较快，但它的缺点是只能做比较单纯的运算，而无法一次做多重且复杂的运算。要想把复杂的运算一次完成，必须靠调用方法，因为方法可以容纳数十甚至数百行表达式，要调用方法，就必须有对象实例来存储方法的相关信息，但是基本数据类型没有足够空间来存储。

Java函数库内提供了8个基本数据类型的类，将基本数据类型包装成相应的对象，并定义许多实用的方法供这些对象调用，这8个类统称为Wrapper类，放在java.lang包中；

在Java5之前，Wrapper类的对象不能直接与基本数据类型计算，必须将对象内的值取出来才可计算。从对象内将值取出称为unboxing(拆箱）；把基本数据类型转换成相应的对象型则称为Boxing(装箱）；

从Java5开始，编译程序会自动做Boxing/Unboxing;

文字与数字的相互转换要用Wrapper类

Java函数库提供了8个基本数据类型的类，将基本数据类型包装成相应的对象，并定义许多实用的方法供这些对象调用，这8个类统称不Wrapper类；
