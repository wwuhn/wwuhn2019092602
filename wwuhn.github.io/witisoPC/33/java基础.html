类、包

包实际上是一组类组成的集合，也称之为类库；

在Java语言中，包的作用不仅可以方便对类实现引用，还可以起规定类方法和变量的被引用范围。通过把具有共同特性的类放到同一个包内，类之间就可以相互访问它们的方法及变量，为成员的访问控制提供了条件。

Java中的访问控制机制和包有密切关系，一般说来包外面的某个类是否访问包内的某个类主要取决于包内这个类的访问级别。而访问级别是由访问控制符决定的。

对象转型

上转型对象

将子类对象的引用赋给父类对象，子类的对象也是父类的对象；（考虑是哺乳动物）；

上转型对象不能使用了父类基础上新增的成员，可以使用重写的方法（信息的部分丢失）；

与上转型对象相似，有接口回调；以接口作参数可以在实现关系中实现多态；

通过继承而产生的类型具有等价性，导出类与基类具有相同的类型，把导出类看作是它的基类的过程称为“向上转型”upcasting,在继承图的典型布局方式中，通常基类在顶部，而导出类在其下部散开，因此，转型为一个基类就是在继承图中向上移动，即“向上转型”。

由于容器只存储Object，所以当将对象引用置入容器时，它必须向上转型为object,因此它会丢失身份。当把它取回时，就获取了一个对Object对象的引用，而不是对置入时的那个类型的对象的引用，这里再度用到了转型，在这里是向下转型为更具体的类型；

向下转型要显性表示；

迭代器方法

iterator()提供了一种通用的方法访问集合中的元素。

迭代器iterator

List cats=new ArrayList();

for(int i=1;i<10;i++){

cats.add(new Cat(i));

Iterator e=cats.iterator();

while(e.hasNext())

((Cat)e.next()).id();

}}

所有集合都有一个iterator()方法，可以返回一个Iterator类的对象（假设取名为e）；

e.next();可以返回序列的下一个元素；

e.hasNext();检查序列中是否还有元素；

e.remove();将迭代器新近返回的元素删除；
