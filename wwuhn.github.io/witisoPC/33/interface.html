To use an interface,you write a class that implements the interface.When an instantiable class implements an interface including signature method and constant type,another company wirte classes to implements the interface.

1 接口只能继承父接口；

2 属性必须赋初值（因为在编译时会自动加上final);

接口其实是抽象类的一种，差别在于抽象类中还可以写一般正常的方法，但是接口内全都是抽象方法。抽象类以定架构为主，但还是有参与一些程序内容的编写；如果定义的是接口，则只是完成了整体架构，程序的内容由其子类用implements接口的结构去实现；

在Java程序设计语言中，接口不是类，而是对类的一组需求描述，这些类要遵从接口描述的统一格式进行定义。

Java是通过接口使得处于不同的层次，甚至互不相关的类可以具有相同的行为。半制品是方法定义（没有实现）和常量变量的集合。在类层次的任何地方都可以使用接口定义一个行为的协议实现它。

在来自不同类的层次的不同部分的类有类似的行为方法。

如，要编写一个SpreadSheet程序，包括含有值的单元的列表集合，希望把String,Float,Dates,Integers,Equations等，都放在SpreadSheet单元中。为做到这一点，String,Float,Dates,Integers,Equations等都要实现同一个方法。要达到这个要求的方式是找到这些类的共同超类，并在超类中实现所需要的方法。但是，由于Object经常是共同的超类，而使这种解决方式不现实。事实上，除了通过Object，所想做到放在SpreadSheet的单元中对象在类层上是无关的，可是Object是不能修改的。事实上，Java提供了一个更好的方式，那就是接口。

Java接口主要用于

1 实现不相关类的相同行为，而不需要考虑这些类之间的层次关系。即捕获不相关类间的类似性，而无须强调类关系；

2 指明多个类需要实现的方法。

3 了解对象的交互界面，而不需要了解对象所对应的类。它展现对象的编程接口，而无须展现对象（这样的对象称为匿名对象）；

接口中的变量（即常量）可以用接口名直接访问；

接口不是类分组结构的一部分，而没有联系的类可以执行相同的接口；

接口把方法的定义和类的层次区分开来，通过它可以在运行时动态地定位所要调用的方法。

修饰         abstract   interface

抽象类

1 抽象类和抽象方法都必须用abstract关键字来修饰；

2 抽象类不能被实例化，也就是不能用new关键字来产生对象；

3 抽象方法只需声明，而无须实现；

4 抽象方法必须定义在抽象类，抽象类的子类必须实现父灰的抽象方法，否则这个子类还是抽象类；

接口

1 接口中声明的变量均为public fianl,所以必须赋初始值；数据成员均为常量；

2 接口没有构造方法；（不需要，因为其属性都有初始值）；

3 接口中定义的方法均为public abstract;

4 不能产生对象实例；

5 子类要实现抽象方法，否则也必须为抽象类；
