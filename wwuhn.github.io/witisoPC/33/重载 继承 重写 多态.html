重载继承重写多态

继承：代码重用；就是重复利用的意思；

重载：同一个类当中编写同名不同参数的方法（参数不一致，调用时根据参数选择）

重写：子类重新定义父类的方法（参数一致，只能调用被重写的子类的方法，如果要调用父类的同名方法，用super关键字），构造函数不能被覆盖；

多态：接口重用，方法重载是实现多性的一种方式；

一名多用：方法的重载；

青出于蓝：方法的覆写；

继承inheritance

继承是指能够直接获得已有的性质和特征，而不必重复定义它们，在面向对象中，继承是子类自动地共享父类中定义的数据和方法的机制。新类继承了原始类的特征，新类称为原始类的派生类（子类），而原始类称为新类的基类（父类）。

继承并不是复制和粘贴，子类并不限制只具有父类提供的状态和方法，还可以定义自己的方法和属性，子类可以覆盖从父类继承来的方法，并为这些方法提供特殊的实现，一个子类只能有一个父类，但是一个父类可以有多个子类，子类具有父类全部的属性和方法。

继承，由类到对象，都是为了减少重复代码；

重载overloading

The Java programming language supports overloading methods,methods within a class can have the same name if they have different parameter list.

overloading:重载：是一个类中多态性的一种表现；

重写overriding

overriding:重写也叫方法的覆盖，子类方法的名称、参数签名、返回值类型必须与父类相同；重写是父类与子类之间多态性的一种表现；

多态polymorphism

polymorphism

多态性意味着一个对象具有多个面孔，可以完成多个不同的功能，从而很好地解决了其他函数型编程语言中为了完成多个不同任务而需要给多个函数定义不同名字的重复化和复杂化。Java是通过方法的重写和重载来实现多态的。

多态：用一个父类类型的变量来引用其直接或间接子类的对象；

在Java语言中，通过方法重写可以在具有继承关系的多个类中定义名称相同但操作不同的多个方法。多态指的正是程序运行时判断执行其中哪个方法的能力；

当一个类有很多子类时，并且这些子类都重写了父类中的某个方法。当把子类创建对象的引用放到一个父类的对象中时，就得到了该对象的一个上转型对象，那么，这个上转的对象在调用这个方法时就具有多种形态，因为不同的子灰在重写父类的方法时产生不同的行为；

多态性就是指父类的某个方法被其子类重写时可以各自产生自己的功能行为；

class Animal{

void cry(){

System.out.println("动物的叫声！！！");}}

class Dog extends Animal{

void cry(){

System.out.println("狗的叫声：汪汪…");}}

class Cat extends Animal{

void cry(){

System.out.println("猫的叫声：喵喵…");}}

class PolymorphismDemo{

public static void makeVoice(Animal a){

a.cry();}

public static void main(String[] args){

Dog dog=new Dog();

Cat cat=new Cat();

//子类的对象也会是父类的对象

makeVoice(dog);

makeVoice(cat);}}

在上面多态的例子中，父类Animal中定义的方法cry()是为了概括其所有子类的一个共同的行为特征，该方法体内的语句并没有实际意义，因为它的所有子类应该覆盖这个方法，所以声明为abstract更合适；

多态也叫动态绑定，动态绑定是指在执行期间（而非编译期间）判定引用对象的实际类型，根据其实际的类型调用相应的方法；为什么叫动态，因为对象只有new出来后才有内存分配，才可绑定相应的方法；其优势是其可扩展性，如果不能动态绑定，则需要增加判定语句去选择相应的方法。如果再增加一个子类对象，重写父类方法，因为是动态绑定（不需判断语句），则不需要更改，同样可以动态绑定方法；

多态存在的三个必要条件

1 要有继承；

2 要有重写；

3 父类引用指向子类对象；

当你调用父类里面被重写的方法的时候，实际当中new的是哪个子类对象即调用哪个方法；

多态通过覆写父类的方法来实现，在运行时根据的对象引用来调用相应的方法；

当我们将子类对象的引用声明为父类的一个对象变量，如果子类有方法，则调用子类的方法，如果子类没有方法，则调用父类的方法；

根据传递对象的引用来调用不同的对象方法；

多态通常意义上指是运行时的多态，而我们有时也称重载为编译时的多态；

接口与多态

当不同的类实现同一个接口时，对接口中的同一个方法可能会有不同的实现，即不同的对象对同一个方法的调用可能会产生不同的行为；

重写父类的方法时，不可以降低方法的访问权限。

子类实现接口的抽象访求也是改写的一种；

方法改写时，父类方法throws抛出的异常范围要能包含子灰方法所抛出的异常范围；

方法的重载对于方法的调用，如计算面积area()的一个方法，如圆，长方形，参数不同，方法体不同，但相同的名称，利于方法调用，采用不同的参数，即可区分调用不同的方法，方法的功能都是计算面积；
