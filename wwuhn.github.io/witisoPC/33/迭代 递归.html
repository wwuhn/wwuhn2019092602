迭代递归

递归过程的执行总是一个过程体未执行完，就带着本次执行的结果又进入另一轮过程体的执行…如此反复，不断深入，直到某次过程的执行遇到终止递归调用的条件成立时，则不深入，而执行本次的过程体余下的部分，然后又返回到上一次调用的过程体中，执行其余下的部分…如此反复，直到回到起始位置上，才最终结果整个递归过程的执行，得到相应的执行结果；

递归过程的程序设计的核心不是对照这种执行流程设计出一种适合“逐步深入，而后又逐步返回”的递归调用模型；

public class Factorial{

public int  f(int n) {

if(n == 0 || n == 1) {   //当n是0或者1的时候，阶乘结果就是0

return 1;

}

else {  //当 n 大于0 的时候，阶乘的结果就是 n*(n-1)*(n-2)* 。。。。

return n*  f(n-1);

}

}

public static void main(String[] args){

Factorial fi=new Factorial();

System.out.println(fi.f(4));

}

}

4*f(3)→4*3*f（2）→4*3*2*1

一个方法可以调用其他的方法，也可以调用其自身，我们可以使用栈帧图来表示方法在调用时的状态。在栈的底部，以参数的最终常数调用自身方法，此时称为基本条件(base case).在基本条件下不会产生递归调用，所以方法不会再有更多的帧；

递归：是在一个方法内部对自身进行调用；
