1. 应具体的知识结构
    1. 系统论思想
      1. 系统的基本思想：整体观点、动态观点、层次观点；
      2. 系统的基本特征：集成、整体、目的、层次、相关、环境适应性；
      3. 系统的核心内容：运筹学管理数学模型、系统管理理论、综合应用方法；
    2. 知识体系架构
      1. 物流工程是关于物流系统分析、设计、改善、控制和管理的学科。
      2. 工程技术、信息科学、物流管理学、系统工程四方面相互交叉；
      3. 以物流系统整体最优为目标：以最低的费用消耗和最高的质量和效率，实现系统整体的综合效益；
    3. 物流规划知识体系：
      从系统的整体利益出发，进行子系统的分析与优化；
  2. 应掌握的内容与技术
    1. 应熟知的内容
      1. 设施规划与设计
      2. 物流运行系统的设计与管理
      3. 物流器具与设备的设计与管理
    2. 应掌握的常用技术
      1. 基础工业工程技术
      2. 系统最优化技术
      3. 分解协调技术
      4. 建模与仿真技术
      5. 网络技术
  3. 应掌握的基本方法
    1. 系统工程方法
      系统工程的三维结构：时间维、逻辑维、知识维；
    2. 系统的运作模式
      系统是把输入转换为输出的转换机构；
    3. 系统分析方法
      系统分析为系统工程的优化提供了逻辑途径；
    4. 相关软系统的方法论
      切克兰德把运筹学、系统工程、系统分析和系统动力学的方法论称为“硬系统”的方法论，自己则提出了一套软系统方法论：为通过比较找到最优方案去解决既定问题；
    5. 应具备处理“二律悖反”的能力
      物流系统与要素以及要素之间的目标会出现冲突或悖反的现象；
      1. 系统内部要素目标冲突
        如运输功能的各运输方式；
      2. 物流服务水平与成本间的制约关系
      3. 物流系统各功能要素之间相互制约关系
        如包装与运输；
      4. 物流系统内外部系统之间冲突关系
  4. 系统规划与设计的分析方法
    1. 系统规划与设计的分析内容
      目标、替代方案、模型、指标体系、评价标准；
    2. 系统规划与设计的综合过程
    3. 系统规划与设计的评价过程
    4. 系统规划与设计的要求与步骤
      1. 确定目标与约束条件
      2. 现状调研、需求分析与方案拟订
      3. 对制订方案进行评价与选择
      4. 方案实施和实施评价
